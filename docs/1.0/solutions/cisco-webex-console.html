Overview

Restcomm Cisco Webex helps service providers enable their Enterprise Customers to send and receive SMS using webexTeams and to manage and monitor the usage

If you want to use the Restcomm Message Exchange feature, you will need to complete the following steps:

  • Create Bot for Enterprise and map the Bot Access Token with Enterprise as explained in section XXX

  • Create a mapping

  • Start sending messages

This document explains Resctomm Cisco Webex Console and how to use features mentioned above.

Home

This is the default page user will see after user logs in. Here user can find the number of enterprises, mappings added and messages sent/received in the last 24 hours within the organisation and 30 days usage report of enterprises, mapping and messages for an organisation

Mappings

To navigate to page, click on the Mappings on the top header.

User can view list of all enterprises that have been added under his organisation. Enterprises can be filtered based on the name.

Enterprise is service providers customers which can help them to understand number of mappings created for specific Enterprise over a specific period of time for billing and management purposes.

Each Enterprise has list of mappings which is basically mapping between SMS enabled number and Cisco WebEx Id.

Adding an enterprise

  • Click on add enterprise button.

  • Enter the name of new enterprise and click add account button

Enterprise name must be unique.

Enterprise mapping Details

  • If user clicks on any enterprise in the list, he will see a paginated list of mappings that belong to the enterprise.

  • Name is the friendly name given to user, number is the external address (SMS enabled number), webexId is the webex teams Id that is mapped with the external number and last column contains the timestamp when the mapping was last updated and the user who updated it.

  • User can edit the name of a particular mapping.

  • User can delete a particular mapping by clicking on the delete icon that appears by hovering mouse on specific row.

Add a mapping

  • User can reach add mapping page by clicking on “Add Mappings” button.

  • User needs to input below 3 fields:

    • Friendly name for the user.

    • External address i.e. the number to be SMS enabled.

    • Webex teams id.

  • User can add single or multiple mappings under a particular enterprise.

  • Webex teams id and number have to be unique across the system.

  • While adding, if all the mappings are valid then user will see the mapping list else user will see the rejected mappings where user can modify the webex teams id or number.

  • When a mapping is added, the number is provisioned on Restcomm and also the number is SMS enabled byTelestax

Search

Enterprise search

The user can perform a search on the enterprises of the organization. Below are the supported filters for enterprise search:

  • Account Scope:

    • Search in Current Account: Shows the list of Enterprises that are created by the logged in user.

    • Search in All Account: Shows the list of all the enterprises under the organisation.

    • Search in Specific Account: To search enterprises that are created by a particular user of the organisation, the user is presented with a list of users of that organisation in a popup window. The user can select the Restcomm account on that popup window.

  • Start Date: Shows the list of enterprises that were created on or after this date.

  • End Date: Shows the list of enterprises that were created on or before this date.

  • Name: Shows the list of enterprises whose name are partially or fully match with the given String:

Mappings search

The user can perform a search on the enterprises of the organization. Below are the supported filters for enterprise search:

  • Account Scope:

    • Search in Current Account: Shows the list of mappings that are created by the logged in user.

    • Search in All Account: Shows the list of all the mappings under the organisation.

    • Search in Specific Account: To search mappings that are created a particular user of the organisation, the user is presented with a list of users of that organisation in a popup window. The user can select the Restcomm account on that popup window.

  • Start Date: Shows the list of mappings that were created on or after this date.

  • End Date: Shows the list of mappings that were created on or before this date.

  • Enterprise: Shows the list of mappings that belong to this enterprise.

  • Name: Shows the list of mappings whose name partially or fully match the entered String.

  • Number: Shows the list of mappings whose external address partially or fully match the entered String.

  • WebexId: Shows the list of mappings whose Webex teams Id partially or fully match the entered String.

Messages search

The user can search the messages of the organization by applying below filters.

  • Start Date: Shows the list of messages that is created on or after this date.

  • End Date: Shows the list of messages that is created on or before this date.

  • Enterprise:Shows the list of messages that belong to this enterprise.

  • To:Shows the list of messages that are sent to this number or WebExId

  • From:Shows the list of messages that are sent from this number or WebExId

  • Direction:Shows the list of messages of particular direction(INBOUND,OUTBOUND). Inbound means messages coming from Mobile to WebEx Id. Outbound means message sent from WebEx Id to Mobile

  • Type:Shows the list of messages of particular type(SMS,MMS). Please note MMS is still not supported.

  • Channel Status:Shows the list of messages with this channel status.

  • Channel Error Code:Shows the list of messages with this Channel Error Code.

  • Channel Error Message:Shows the list of messages with this Channel error message.
