Overview

Restcomm Cisco Webex helps service providers enable their Enterprise customers to send and receive SMS using Webex Teams and to manage and monitor their usage. Enterprise customers can send two-way SMS.

If you want to use the Restcomm Cisco Webex feature, you will need to complete the following steps:

Create Bot for each of the Enterprises and map the Bot Access Token with Enterprise as explained in 4 Create Bot for Enterprise Create a mapping (Via API or UI Console) Start sending messages

Prerequisite

In order to complete the aforementioned steps, you will need:

  • Webex setup for your Enterprise Customers

  • Working knowledge of cURL and a way to make HTTP API calls (either using cURL or Postman; we recommend Postman if you’re not familiar with cURL.) Or you can directly use UI Console exposed by Restcomm.

  • An understanding of Restcomm Message Exchange API’s for Cisco Webex or Console on how to add mappings.

  • An activated Restcomm Cloud account.

Setting up your first mapping

You must have a valid, working Restcomm account in order to complete this step. If you do not have one, you can contact us.

The purpose of Webex mapping is to associate a phone phone number with an existing Cisco WebexId. For example, imagine you have a user with the Webex teams id of joe@myserver.com and Restcomm number is +1-919-555-1212.

Mapping will tell the Restcomm Message Exchange service for Cisco Webex to send all inbound SMS messages sent to Joe’s number to joe@myserver.com. Additionally, any time Joe sends an outbound SMS from his Cisco Webex account, it will be sent from his phone number.

You can set up your mapping in two different ways, either via the Restcomm Message Exchange Console for CiscoWebex or via the API.

Setting up a new mapping via Console

  • Log in to your Restcomm Message Exchange Console via your Restcomm cloud account

  • Create new Mapping as explained in Section 2.3 Add Mappings in Docs :Cisco Webex console

  • Before creating mapping, you need to create Enterprise. This is a very important step as Cisco Webex Bot has one to one mapping with Enterprise.

  • Create Bot (for each enterprise) as explained in 3 Create Bot for Enterprise

Receiving an SMS/message in Webex teams

Once you’ve completed all of the steps above, you’re ready to test.

Send a message from PSTN number to mapped number .You should then see the message appear in Webex teams whose Cisco Id is mapped with this restcomm number. The message should appear in new Webex Space if you are receiving a message for the first time. All subsequent messages appear in the same space. Space name by default is phone number from where message was initiated. You can rename space to something more meaningful, for example, “Ryan’s Mobile Phone”

Replying to received SMS/message

To reply to received message follow steps below:

  • In newly created space, refer to Bot with “@” prefix and then type your message

  • Hit Enter and your message will be sent as SMS to mobile number of receiver.

Sending an SMS/message to new Number from Webex Teams

To send SMS to new number, follow below steps:

  • Add newly created Bot as Contact in your Cisco Webex if not done earlier. This is one time process.

  • Send any private message to this Bot

  • It should respond back with Card that allows you to enter the receiving party’s phone number

  • Enter receiving party’s phone number and hit Submit. It should create a new Space with receiver Phone number as Space name

  • In newly created space, refer to Bot with “@” prefix and then type your message

  • Hit Enter and your message will be sent as SMS to mobile number of receiver.

Create Bot for Enterprise

  • Visit https://developer.webex.com/ then click on “Start Building Apps”.

  • Login with the webex teams for developers credentials.

  • Click on “Create a New App”.

  • Click on “Create a Bot”.

  • Fill the basic details of the bot such as “Bot Name”, “Bot Username”, “Icon”, “Description” then click “Add Bot”.

  • You will get “Bot’s Access Token”, “Bot Id”

Mapping Bot Access Token With the enterprise

Once bot is created in Webex we will get bot access token To map Bot with an enterprise we have to call the below api.

curl -X POST https://mycompany.restcomm.com/restcomm/2012-04-24/webex/webexbot/access.json \
   -H 'Cache-Control: no-cache' \
   -H 'Content-Type: application/json' \
