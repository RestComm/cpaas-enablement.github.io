Introduction

Two-factor authentication (2FA) API lets you send One Time Password (OTP) using SMS, Voice or email.

2FA allows you to add a second layer of security to your Application in addition to the user-id and password. The flexibility of using any channel makes sure your users get OTP even when they experience issues with one of the channels.

This document covers features and functionalities of 2FA UI Console that leverages underlying API.

To access 2FA Console visit https://<your_custom_restcomm_domain>/twofa/ and login with your Restcomm Credentials.

White Labeling

Restcomm enables you to go to market with your own 2FA brand in a matter of days. 2FA leverages the Restcomm Whitelabel, hence all the work done by user to whitelabel Restcomm Dashboard is used by 2FA as it is. For further details on Restcomm Dashboard whitelabel please visit https://www.restcomm.com/docs/whitelabeling.html

Home

This is the default page user will see after the user logs in. Here user can find the number of OTP requests sent per channel - SMS, Voice and Email in the last 24 hours for users Account and 30 days usage report of OTP requests.

Session Details

Session Details Records allows users to search for a specific set of 2FA request based on filter criteria or to retrieve Usage Records like how many 2FA requests per day based on filter criteria.

Search

The user can search for a specific set of 2FA. Below are the supported filters for search:

Account Scope:

  • Search in Current Account: Shows the 2FA requests that are created by the logged-in user.

  • Search current account and sub-accounts: Shows 2FA requests that are created by logged in user and all other Sub-Accounts under logged in user.

  • Search in Specific Account: Search 2FA requests created by specific Sub-Account under logged in user. The user is presented with a list of Sub-Accounts in a popup window. The user can select the Restcomm account on that popup window.

Start Date: Shows the list of 2FA requests that were created on or after this date.

End Date: Shows the list of 2FA requests that were created on or before this date.

Status: Show the list of 2FA requests that have selected status.

Channel: Show the list of 2FA requests that are sent via the selected channel.

Service: Show the list of 2FA requests that match this service text, partially or fully in any position.

From: Only include 2FA list that has from begins with this number/email text. For example, if OTP is generated from number “15713316943” and from filter parameter is “1”, it will match. However if from filter parameter is “3”, this record will not match. This allows filtering 2FA sent from specific country code.

To: Only include the OTP list that has to begin with this number/email text. For example, if OTP is sent to number “16513333367” and to filter parameter is “1”, it will match. However if to filter parameter is “3”, this record will not match. This allows to filter OTP sent to specific country code.

TargetSid: Only include 2FA list that match target sid of SMS/ Call/ Email text, partially or fully in any position.

Check Status: Only include a 2FA list that has this check status. Possible values are valid or invalid

Channel Status: Only include 2FA list that matches underlying channel status, partially or fully in any position. For SMS, possible channel status value can be as explained here. For Call, possible values are as explained in “status” property here.

Channel Error Code: Only include a 2FA list that matches underlying channel code, partially or fully in any position. For SMS, the possible value can be found here

Channel Error Message: Only include 2FA list that matches channel error message text, partially or fully in any position

Sorting

Users can sort the search results by clicking on Column Header as explained below.

Date: Ascending or Descending list of 2FA based on Date when it was created Service: Sort by Service. Status: Sort by the status of 2FA request.

Download as CSV:

Once results are displayed for search criteria, the user can click on the Download icon below the Search icon to download search results as CSV.

2FA request details:

Users can view details of particular 2FA by clicking any row in the list.

Users can click on Check link to see the details of the 2FA check request

User can click on Events to see details of Channel used to send 2FA request.

Usage Records

The Usage record allows to retrieve usage made by your 2FA account during any time period and by any usage channel. Below filter parameters are supported

Account Scope:

  • Search in Current Account: Shows the 2FA requests that are created by the logged-in user.

  • Search current account and sub-accounts: Shows 2FA requests that are created by logged in user and all other Sub-Accounts under logged in user.

  • Search in Specific Account: Search 2FA request that is created by specific Sub-Account under logged in user. The user is presented with a list of Sub-Accounts in a popup window. The user can select the Restcomm account on that popup window.

Subresource:

  • Daily: Shows the record of 2FA on a daily basis.

  • Monthly: Shows the record of 2FA on a monthly basis.

  • Yearly: Shows the record of enterprise on a yearly basis.

  • Today: Shows the record of enterprise today.

  • Yesterday: Shows the record of enterprise on yesterday.

  • This Month: Shows the record of enterprise this month.

  • Last Month: Shows the record of enterprise last month.

Start Date: Only include 2FA requests that were created on or after this date.

End Date: Only include 2FA requests that were created on or before this date.

Status: Only include 2FA requests that have selected status.

Channel: Only include 2FA requests that are sent via selected channel.

Service: Only include 2FA requests that match this service text, partially or fully in any position.

From: Only include 2FA requests that has from begins with this number/email text. For example, if OTP is generated from number “15713316943” and from filter parameter is “1”, it will match. However if from filter parameter is “3”, this record will not match. This allows filtering 2FA sent from specific country code.

To: Only include 2FA requests that have to begin with this number/email text. For example, if OTP sent to number “16513333367” and to filter parameter is “1”, it will match. However if to filter parameter is “3”, this record will not match. This allows filtering OTP sent to specific country code.

Check Status: Only include 2FA requests that have this check status. Possible values are valid or invalid

Channel Status: Only include 2FA requests that match underlying channel status, partially or fully in any position. For SMS, possible channel status value can be as explained here. For Call, possible values are as explained in “status” property here.
