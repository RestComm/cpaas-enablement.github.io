Number Masking

Introduction

NumberMasking exposes API to obfuscate called/calling parties. As of today API allows only voice conversation but will soon be followed by SMS as well.

Number Masking allows users to create and manage:

  • Application

  • Mask Number Pool

  • Context

  • Participants

  • Interactions

Developers can create Applications for example Taxi Booking Application or Food Delivery Application and for each Application assign Phone Numbers made available by your Service Provider to create a pool of Mask Numbers. Whenever a ride is booked by Customer, the developer will create a new Context and add Participants (driver and customer) to that Context. Number Masking will automatically select the Mask Number from the pool and assign to Participants. The Mask Number is shared with respective Participants. Whenever a Participant calls Mask Number, other Participant in Context will receive a call from their assigned Mask Number thus maintaining complete privacy.

There can be zero, one or many interactions between Participants till the time Context is alive.

Application

Application is a simple way to distinguish separate number masking applications. Applications have Context and each Context has Participants. Applications also have Numbers within it that are used for masking

Mask Numbers

For Number Masking to work you have to associate at least one Phone Number to an Application. Once a number is added to the Application, it is called a Mask Number. All added numbers go into the Mask Number Pool associated with a given Application. Once a Phone Number is associated with Application, it cannot be assigned to another Application.

Phone numbers can be marked as Reserved. Reserved numbers will not be included in the set of numbers Application considers when choosing a number for a participant, but they can be manually assigned as Mask Numbers for participants.

Context

A Context is a single instance of two individuals communicating. It belongs to an Application and maps two Participants. A Context can be set for example between Taxi Driver and Customer and it can be active till a predefined period of time or can be deactivated upon request via API call.

Context State

Context can be in below states at any given point in time.

Open: Context is Open which means it has yet not passed expiry date or ttl. But there are no active communications in this Context

In-Progress: There are active communications going on in this Context. This is more meaningful for Voice where Context can remain In-Progress till the call is on-going between participants.

Closed: Context has either passed expiryDate or ttl. No more Communication is possible in this state.

Participants

A Participant is a single individual communicating in a Context. There is a limit of two Participants per Context.

You can create a Participant in two ways:

By specifying the private Phone Number of the Participant (participantPhoneNumber) and relying on Number Masking to select an appropriate Phone Number from the Mask Number pool. The Mask Number returned (maskPhoneNumber) is the Phone Number that the Participant should call to reach the other party.

By specifying both the participantPhoneNumber and the maskPhoneNumber. In this case, Number Masking will attempt to use the specified Mask Phone Number provided. Note: The Mask Phone Number specified must already be added to the Mask Number pool in order to be accepted.
