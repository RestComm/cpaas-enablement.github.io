Number Masking Console

Overview

Number Masking enables Enterprise to easily make communication between two or more parties without revealing private information (Phone Number).

Enterprise customers can define, manage and view Applications, Contexts, Mask Number Pool etc.

This document covers the features and functionalities of the Number Masking Console (UI) from Enterprise user Point of View.

To understand more on API please refer to Documentation here.

To sign in to Console please visit https://<domain-provided-by-csp>/numbermaskingconsole/

Applications

To navigate to the page, click on the Applications on the top header.

Users can view a list of all Applications that have been created under Enterprise.

Applications can be filtered based on the name.

Add Application

To create a new Application:

  • click on Add Application button

  • enter the name of the application and click the Done button.

Name of the application should be unique.

Basic details

To edit an Application click on the name of the Application in the list of applications, users will view the Basic Details page.On the page, enter the fields and click on the Save button.

Description of the fields to edit application:

Default Time to Live: Default time to live for context in the application after Contexts last update date or interaction. If not passed, default is 0 which is unlimited

Application Name: Name of the application.

Mask Numbers

To navigate to this page click on Mask Numbers in the side navigation bar of the screen.

Users can view a list of all Mask Numbers in the pool that is assigned to the Application.

Mask Numbers list can be filtered based on the mask number.

To mark a Mask Number as reserved/unreserved click on checkbox of particular mask number

Add Mask number

To add a mask number click on the Add Mask number button.

List of all the Phone Numbers belonging to the Enterprise and not assigned to any Applications are shown. To add a Phone Number as Mask Number, user need to select the checkbox of particular number.

Context

To navigate to this page click on Contexts in the side navigation bar of the screen.

Users can view a list of all Contexts that have been created under the Application.

Contexts can be filtered based on the name of the Context.

Add Context

To add a context click on the Add Context button.

Enter the fields required and click on the Done button.

Description of the field to add context:

  • Context Name: Mandatory field and should be unique within Application.

  • Time to live: Optional field. If provided, it overrides the default time to live configured in the basic details of application .

  • Expiry Date: Optional field,the date when context should expire, this will override time to live if configured.

  • Mode: Mode of the interaction, example:Voice.

Edit Context

To edit a context click on the context name in the contexts list page. To go back to the context list page click on the back button on top right.

Users can view the details of the context.

Can edit context by updating Name, Expiry date, Mode, Status, Time to live and click on the Save Changes button.

Can Delete the context by clicking on the Delete button

Participants

To view or to add the participants in the context click on participant link of particular context in context list page

To go back to the context list page click on the back button on top right.

Users can view the list of participants which are added under that particular context.

Participants can be filtered based on the participant number.

Add Participants

To add a participant in the context click on the Add Participants button. Enter the fields required and click on the Done button. Description of the fields to add participant:

  • Participant Name: Optional. Name of the participant. * Phone number: Mandatory field. Phone number of participant

  • Mask Number: Optional field. Mask Number available in Mask Number pool of Application.

  • Mask Number SID: Optional field. Sid of the Mask Number number.

If both Mask Number Sid and Mask Number are specified ,Mask Number SID takes preference. NOTE: If both Mask number and Mask Number SID are not specified, Application will select a random number from the Mask numbers pool.

Participants Details

Click on the name of the participant to view the details of the participant.

Numbers

Your Communications Service Provider (CSP) would have reserved Phone Numbers for your Enterprise to use. Only these numbers can be used as Mask Numbers.

To navigate to the page, click on the Numbers on the top header.

Users can view a list of all numbers that have been assigned for your Enterprise.

Numbers can be filtered based on the phone numbers.

User Management

To navigate to the page, click on the User Management on the top header.

Users can view a list of all users that have been created under enterprise.

List of users can be filtered based on the user name.

Add Users

To add users under enterprise click on the Add User button in the User management page.

Follow the steps below to add an user:

  • Enter the name of the new user.

  • Enter email id of the user.

  • Enter the password for the user account.

  • Set a role for the user (Enterprise ADMIN)

  • Set the status of the account(ACTIVE,INACTIVE)

Mail ID must be unique. NOTE: Password should meet below criteria: At least one Uppercase letter(A,B,C,D…​) At least one Lowercase letter(a,b,c,d,…​…​.) At least one number(1,2,3,4,5.) At least one special character(!,@,#,$,%,&,…​…​.) Length of the password should be greater than equal to 8.

User Details

Click on the name of the user in the users list to view the details of the user.

Click on Close Account to delete the user

Search

Search Application

To navigate to the page, click on the Search on the top header.

The user can perform a search on the applications of the enterprise. Below are the supported filters for application search:

  • Start Date: Shows the list of applications that were created on or after this date.

  • End Date: Shows the list of applications that were created on or before this date.

  • Application Name: Shows the list of applications whose name are partially or fully match with the given name:

Application Details

Users can view details of a particular application by clicking any application name in the list.

Search Mask Numbers

To navigate to the page, click on the Mask numbers on the side navigation bar..

users can perform a search on the Mask numbers of the enterprise. Below are the supported filters for Mask number search:

  • Start Date: Shows the list of Mask numbers that were created on or after this date.

  • End Date: Shows the list of Mask numbers that were created on or before this date.

  • Application: Shows the list of Mask numbers that belong to the application selected from the dropdown.

  • Is Reserved: Shows the list of Mask numbers with this Is Reserved status(True, False).

  • In Use: Shows the list of Mask numbers with this In Use status (True, False).

  • Mask number: Only show Mask Number that starts with phoneNumber digits. This is useful to search Mask Numbers that belong to specific Country Code.

Mask Number Details

Users can view details of a particular mask number by clicking any sid in the list.

Search Context

To navigate to the page, click on the Contexts on the side navigation bar..

Users can perform a search on the Contexts of the enterprise. Below are the supported filters for Context search:

  • Start Date: Shows the list of Contexts that were created on or after this date.

  • End Date: Shows the list of Contexts that were created on or before this date. A* pplication: Shows the list of Contexts that belong to the application selected from the dropdown.

  • Status: Shows the list of Contexts that matches with this status (Open, In progress , Closed)

  • Context name: Shows the list of Contexts whose name are partially or fully match with the given name:

Context Details

Users can view details of a particular Context by clicking any Context name in the list.

Search Call logs

To navigate to the page, click on the Call Logs on the side navigation bar.. The user can perform a search on the Call Logs of the enterprise. Below are the supported filters for Call Logs search:

  • Account Scope:

    • Search in Current Account: Shows the list of call logs that are created by the logged in user.

    • Search in current Account and sub accounts : Shows the list of all the call logs that are created by the logged in user and their sub accounts.

    • Search in Specific sub Account: To search call logs that are created by a particular sub account user, the user is presented with a list of sub account users in a popup window. The user can select the subaccount on that popup window.

  • Start Date: Shows the list of call logs that were created on or after this date.

  • End Date: Shows the list of call logs that were created on or before this date.

  • Application: Shows the list of call logs that belong to the application selected from the dropdown.

  • Context SID: Shows the list of call logs whose Context SID partially or fully matches the entered context sid.

  • Inbound participant Number: Shows the list of call logs whose Inbound participant Number partially or fully matches the entered number.

  • Inbound Mask Number: Shows the list of call logs whose Inbound Mask Number partially or fully matches the entered number.

  • Outbound Participant Number: Shows the list of call logs whose Outbound Participant Number partially or fully matches the entered number.

  • Outbound Mask Number: Shows the list of call logs whose Outbound Participant Number partially or fully matches the entered number.

Call log Details:

Users can view details of a particular Call log by clicking any SID in the list.

Usage Records

The Usage record REST resource provides a simple API to retrieve usage records of your Number Masking application during any period of time and in any mode (Voice, Message or VoiceAndMessage). This makes it easy to build reporting and analytics tools for your application.

Usage Records for Number Masking consist of Contexts created, Voice minutes including number of calls, and SMS Units.

Usage Records data is collected every 15 minutes. So there is possibility that once Context/Interaction is completed, it may take maximum 15 minutes to be reflected in the Usage Records.

Context

To navigate to Context click on the Usage Records on top menu and navigate to Context.

Below is a list of all supported filters you can use to search Context Usage Records.

  • Subresource: The main usage records list resource supports a variety of convenient subresources like Daily, Monthly etc. If you would like to see consolidated usage records for specific time period, avoid using this filter.

  • Start Date: List usage records starting from the specified start date filter.

  • End Date: List usage records ending at the specified end date filter.

  • Application: List usage records for the specified Application filter.

  • Category: List usage records for the specified Context mode.

Interactions

To navigate to Interactions click on Usage Records → Interactions

Below are supported filters you can apply to search Interactions Usage Records

  • Sub Resource: The main usage records list resource supports a variety of convenient subresources like Daily, Monthly etc. If you would like to see consolidated usage records for specific time period, avoid using this filter

  • Start Date: List usage records starting from the specified start date filter.

  • End Date: List usage records ending at the specified end date filter.

  • Application: List usage records for the specified Application filter.

  • Category: List usage records for the specified Context mode.
