Overview

This document features the product information and the necessary steps to get started with CPaaS for SMS enablement of Microsoft Teams.

Prerequisite

You should have a registered Microsoft Azure account. You should also be the administrator for your organization’s in the Microsoft Teams platform.

To begin, you will need to obtain the values of the following parameters associated with Microsoft Teams.

  • Application ID

  • Tenant ID

  • Client Secret

  • Bot App ID

Application ID & Tenant ID

Once this step is completed you will have Applications and Tenant ID available.

Follow these steps to register your new application.

  • Sign in to the Azure Portal using your Microsoft Teams credentials.

  • Navigate to "Azure Active Directory" by clicking on the menu at the top left corner as shown below.

  • Click on "App registrations" and create a new application by clicking on the "New registration" link in the horizontal menu as shown below.

  • The registration screen will appear as depicted below:

  • Enter a name for the application. Your Microsoft Teams users will see this name, so it is best to choose a name that is meaningful such as "SMSApp". You can change it later.

  • For "Supported account type" (Specify who can use the application, sometimes referred to as the sign-in audience)please select Option 3 "Accounts in any organizational directory and personal Microsoft account"

  • "Redirect URI" is optional and doesn’t need any value.

  • Click on the "Register" button

Getting Application ID

Once you have successfully created the application, the overview screen will be displayed as shown below. Here you will get the Application ID:

Getting Tenant ID

In the same screen that is mentioned above, you will get the Tenant ID.

Add Permissions to the Application

To add permissions to the application, follow these steps.

  • Click on the API permissions from the left navigation and select Add a permission as shown below.

  • From the available options, select the Microsoft Graph section.

  • From the Microsoft Graph Permissions sections, select the Application Permission.

  • Once Application permission is selected, specific permissions need to be granted to the application. From the search textbox, search for the following permissions and select individually.

    • User.ReadWrite.All

    • Directory.ReadWrite.All

    • Group.ReadWrite.All

    • TeamsAppInstallation.ReadWriteForTeam.All

  • Once the permissions are selected, click on the Add Permissions button. Once permissions are granted, the following screen will be shown.

  • All the permissions require a tenant admin to consent to these permissions. To proceed, click to Grant Admin consent as highlighted in the previous screenshot and select the "Yes" option.

  • Once admin consent is granted the permission screen will look like this.

Client Secret

In this step, we will set up a "client secret". Please follow steps below

  • Select Certificates & secrets then click to create a New client secret.

  • Add the description of the client secret and also select the expiry as "Never" as shown below. The description can be any meaningful phrase. Click on Add button.

Once you add the client secret, the value is shown one time only. Be sure to copy it (you may use the copy-to-clipboard option) and store it somewhere safe.

Create a Bot

To create a Bot Channel Registration please follow steps below

  • Navigate to the Azure portal home page.

  • Click to create a resource.

  • In the search text box type "bot" and hit enter as shown.

  • From the available options, select the Azure Bot.

  • Click the Create button on the Azure Bot page.

  • The previous step will lead you to the Azure Bot form. Enter the requested information about your bot as shown below:

    NOTE

    If the registration form does not open and you are prompted to create a free account then follow the steps outlined in the Access Management for Azure Resources section.

  • Bot handle: This is a unique name for your Bot, in order to distinguish it from other bots inside the same account. You can give it a name of your choice.

  • Subscription: Keep this as default.

  • Resource group: you can create a new resource group or you can choose an existing one from the list.

  • New resource group Location: Select "East US" from the drop down menu.

  • Pricing tier: Click on “change plan” and select F0 Free as shown below:

For this option, There are no restrictions on the number of messages that can be sent. As our solution is using “Standard channel” there will be no overhead charges.
Additional link explaining the Azure Bot Service pricing: https://azure.microsoft.com/en-us/pricing/details/bot-services/

  • Microsoft App ID: Choose the “Use existing app registration”

  • Under the ”Use existing app registration”, fill in the following details and press the Ok button as shown in the following screenshot.

    • Existing app id: Enter "Application ID" as explained in the previous section.

    • Existing app password: Enter the "Client secret" as explained in the previous section.

  • Click on the Review + Create button. Once validation has passed you will be redirected to the following page:

  • Click on the Create Button

  • After the Bot Channel Registration is completed, you’ll need to enable the Teams channel. You will need to follow these step in order to link the Microsoft Teams Application with our Bot.

    • Navigate to the Azure portal home page. Under Azure services, click on Resource groups. The resource group page should list the resource you created while setting up your Azure Bot. In the example above, the Resource group was named "messagetestbot". Click on this resource. It should list the Bot created in the previous section. Click on the bot that you just created. After this step your bot will appear under Recent resources in the Home Page as depicted below.

  • Once you click on the name of the Bot, you will be redirected to a new page from which, you will need to choose the Channels option from the left menu.

  • In the Connect to channels screen, click on the Microsoft Teams icon.

  • Once you click on the Microsoft Teams Icon, keep everything default and hit Save.

It will display the Terms of Service, to which you must agree to proceed.

  • Once the Microsoft Teams channel is configured, you will need to add the “messaging endpoint”. To do that, click on the name of the Bot. You will be redirected to a new page. Click on the Configuration option from the left menu.

  • You will see a screen with a Configuration form. Enter the following URL in the “Messaging endpoint” field: https://$DOMAIN/msteams/internal/message

  • Then click on Apply button.

Add Bot to Microsoft Teams

For adding the Bot to Microsoft Teams, you will need a Zip file of the application which you can find it here

You will need to Unzip the file. Once you unzip the file you will see three different files in it.

Now you will need to make some changes in Manifest.json file:

  • Replace the <BOT-NAME> field by the name of your choice which will be reflected as the Bot name. (There are 5 replacements for this field ).

Replace << >> as well.

  • You will need to replace the Application ID field with your Application ID. There are two replacements for this field.

Once you make the changes it will look like this:

After the previous steps are completed, you will need to select all the three files which are present in the folder and convert it into a Zip file.

Once the zip file is ready, you will need to go to the Microsoft Teams Application. Navigate to the Apps section from the bottom left side of the screen and select the Upload a Custom App.

For Uploading the Application to Microsoft Teams the user who is trying to upload the App, must have a Global Administrator role assigned to them on Microsoft Azure.

Upload the zip file which you have created in the last step.

Once you have successfully added the Bot to Microsoft Teams, you will find it under the Apps section. It will show up with the name you chose in the previous step.

Getting Bot App ID

Once you have added the Bot to Microsoft Teams, you will find it under the name you chose in the manifest file. Follow the steps below:

  • Hover your mouse on the newly added Bot, and then click on the three dots.

You will see three options, from which you will need to click on the Copy Link. Once you do so it will be copied to the clipboard. From that you will need to pass on the Bot App ID. The link will look similar to the one below:

https://teams.microsoft.com/l/app{Bot app Id}?source=store-copy-link

Example: https://teams.microsoft.com/l/app/f76e7e66-c5ed-4d37-a7bc-062f7be377a8?source=store-copy-link.

Sending and receiving Messages/Sms

Once you have a mapping registered with a phone number and a Microsoft Address, you are ready to send and receive SMS on Microsoft Teams.

Receiving an SMS/message in Microsoft teams

If there is an Incoming SMS for the mapped number, you should see the message appear in the Microsoft Teams Team. The message should appear in the new Team if you are receiving a message for the first time. All subsequent messages appear in the same Team. The Team name by default is the phone number from which the message was initiated (sender). You can rename this space to something more meaningful, for example, "Jane’s Mobile Phone"

Sending an SMS/Message to New Number from Microsoft Teams

To send an SMS to new number, follow below steps

  • Start a conversation with the newly created Bot. You will find the Bot in the App’s section and you will need to click on the Bot’s name

  • Click the Open button. As depicted in the screenshot below:

  • Once you click Open, you will need to send any private message to this Bot.

  • It should respond back with a Card that allows you to enter the receiving party’s phone number

  • Enter the receiving party’s phone number and hit Submit.

  • Once you submit, it should create a new Team with the receiver’s phone number as the Team name.

Sending a Message

For sending a message to the receiver’s phone, you can either reply back to the messages or you can start a new conversation.

  • Option 1: Replying back to the message.

    • Click on the name of the Team to which you need to send a message

  • Once you are inside the Team, click Reply.

Once you click Reply, search for the Bot’s name with the "@" prefix and select the Bot from the list of available options.

Once you select the bot, type in the message that you want to send and hit enter.

Your message will be sent as SMS to the mobile number of the receiver.

  • Option 2: Starting a new conversation.

    • Click on the name of the Team to which you need to send a message.

  • Once you are inside the Team, click on New conversation.

  • Once you click on the New conversation, search for the Bot’s name with the "@" prefix and select the Bot from the list of available options.

  • Once you select the bot, type in the message that you want to send and hit enter.

  • Once you hit enter, your message will be sent as SMS to the mobile number of the receiver.
