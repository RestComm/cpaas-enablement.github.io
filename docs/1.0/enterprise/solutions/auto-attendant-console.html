Overview

Auto Attendant (AA) is intended for business customers who want to provide an automated means of directing incoming calls to the appropriate person within the organization, typically to offload this task from a receptionist and also offer a simple menu system ("for menu, press 1, for transfer to phone, press 2," etc.)

Restcomm Auto Attendant helps enterprise users to Edit Autoattendant,manage users,assign phone numbers and assign extensions. Enterprise Customers can define different menus for given AA. For example

  • Work hour menu provides option to caller to reach to out Extension etc

  • Non work hours menu can request to call back during working hours

  • Special day menu can announce different greetings

It allows Enterprise Users to create menus,schedules,announcements for an auto attendant system.

This document covers the features and functionalities of the Telestax Auto Attendant from Enterprise user Point of View.

Auto Attendant System

AA system List

To navigate to the page, click on the Systems on the top header.

  • Users can view a list of all AA systems that have been added under enterprise.

  • AA System can be filtered based on the name.

  • User can deactivate/activate the aa system by clicking on the status of any AA system

AA system

To edit an existing AA system click on the name of the AA system. Both of the actions will take the user to the basic details page.

Steps to follow while creating an AA system.

  • Basic details

  • Announcement

  • Menu

  • Schedule

  • Special Days

  • Error Handling

Basic details

  • Add a name of the AA System.

  • Assign the phone numbers from the list of available phone numbers.

  • Assign the extension numbers from the list of available extensions,

  • Set the time zone where the AA system is being used.

  • Add the description to the AA system

  • Click Save & Continue Button

Announcement

To navigate to this page click on announcements in the side navigation bar of the screen. Users can view a list of all announcements that have been added under the enterprise.

Create Announcement

Click on Create Announcement button.

  • Enter the name of the announcement.

  • Enter description of the announcement.

  • User can upload a announcement by:

    • Click on the Record button and record the announcement through the browser itself.

    • Drop the wav file with 8 khz mono specification from the local files into the specified field.

    • Click on Browse and select the audio file of announcement from the local files (wav file with 8khz mono specification)

Menu

To navigate to this page click on Schedule in the side navigation bar of the screen.

Below the schedule table users can view the list of menus available for that AA system

Users can change the color code of the menu by clicking on the color box in the list to change the color of the menu and click on select. Users can delete the menu by clicking on the delete icon of the particular menu.

Note: Menu can only be deleted if it is not used in schedule and there is no announcement assigned to it.

Create menu

To add a menu click on Add Menu button.

  • A popup screen will appear:

  • Enter the name of the menu.

  • Assign an announcement to the menu from the list of the announcements.

  • Assign action to be performed for the keys mentioned on the screen and respective data of the action if needed to carry out that particular action. Please see section 4.2.3.2 Menu Actions to understand the list of actions possible.

  • Set the timeout of the menu in seconds and set the action and respective action data (if needed) to be performed after the timeout.

  • Menu will be created by clicking on the Done button.

After series of DTMF pressed by caller, its expected that user presses # key to indicate to the system that the caller is done with input. If user doesnt press # key, System waits for timeout seconds and then carries necessary action depending on DTMF input.

Menu Actions

Table below explains each of Menu actions that can be assigned to keys.

Menu Action

Description

Announcement Hangup

Play the selected Announcement and hang up the call

Announcement Return

Play the selected Announcement and return to the menu. System will play the Menu Announcement again.

Dial By Extension

Dial the extension number entered by caller

Dial By Name

Listen to speech from the caller and dial the corresponding extension. For this to work correctly the Name must have been defined for extensions.

Go To Menu

Play the selected Menu

Hangup

Hang Up the call

Replay Menu

Play the existing Menu again

Return To Previous Menu

Play the previous Menu again

Transfer To Phone

Transfer the call to specified Phone Number

Schedule

To navigate to this page click on schedule on the side navigation bar of the screen .

  • A grid will appear where you can create a weekly schedule for different menus.

  • Select a menu checkbox, drag and create the schedule for it in the grid.

  • Grid can be changed to a 15 minutes or 30 minutes time window.

Special Days

To navigate to this page click on special days on the side navigation bar of the screen .

  • Select a checkbox of menu from the list of menus below the calendar

  • A calendar will appear where you can create a special days schedule from different menus.

  • Select a menu checkbox, select a date on the calendar to create special day schedule for it .

Error Handling

To navigate to this page click on Error Handling on the side navigation bar of the screen .

In each AA System that you create, you should always define the action to be taken if the menu times out or the user presses invalid key or the user presses invalid extension: For example,if the user does not press a key within the specified time. you could repeat the menu so that the caller has another chance to check the options and select the right key, or transfer the caller to an operator.

Timeout

If the caller doesnt press any DTMF for configured timeout duration, system will perform the configured action.

  • Set the timeout seconds to wait for the schedule.

  • Set how many consecutive timeout occurrences are allowed after which

  • Set the action and respective data of the action if needed to carry out that particular action.

If timeout is defined at Menu Level, it will override the global timeout defined here
Unknown Input

If the menu detects unknown input : that is, if the user presses a key which is not defined in the menu .

  • Set how many occurences that the user presses unknown input after which

  • Set the action and respective data of the action if needed to carry out that particular action.

Invalid Extension

Set the action to take place If the user presses a invalid extension which is not configured in the aa system .

  • Set how many occurences that the user presses invalid extension after which

  • Set the action and respective data of the action if needed to carry out that particular action.

Phone Numbers

One or more lines/phone numbers are setup as Auto Attendant phone numbers. This means that the system provides Auto Attendant Service to caller on these numbers.

Phone numbers List

To navigate to the page, click on the phone numbers on the top header.

  • Users can view the list of phone numbers under the enterprise.

  • Users can assign the phone numbers to the AA system by selecting the AA system from the drop down of AA system names.

Extensions

Extension List

To navigate to the page, click on the Extensions on the top header.

  • Users can view the list of Extensions that are configured under the enterprise.

  • Enterprise Admin can edit the extension number and spoken name of the number by clicking on the pencil icon in the respective fields( Spoken name, Extension)

  • AA Admin can edit the spoken name by clicking on the pencil icon

User Management

Users List

To navigate to the page, click on the User Management on the top right dropdown appears on clicking name of the user.

  • Users can view the list of accounts registered under the enterprise.

  • Users can deactivate/activate the account by clicking on the status of any account.

All the users created are Sub-Accounts under that CSP Admin Account in Restcomm

Add Users

Follow the steps below to add an user:

  • Click on the Add User button in the User management page .

  • Enter the name of the new user.

  • Enter email id of the user.

  • Enter the password for the user account. This will be set in RC.

  • Set a role for the user (BG ADMIN,AA ADMIN)

  • Set the status of the account(ACTIVE,INACTIVE)

Mail ID must be unique. Password should meet below criteria:

  • At least one Uppercase letter(A,B,C,D…​)

  • At least one Lowercase letter(a,b,c,d,…​…​.)

  • At least one number(1,2,3,4,5.)

  • At least one special character(!,@,#,$,%,&,…​…​.)

Length of the password should be greater than equal to 8.
