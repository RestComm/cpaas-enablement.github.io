Auto Attendant API

Overview

Telestax Auto Attendant (AA) allows Communications Providers (CPs) to deliver its Business Customers (also known as Enterprise) automated means of directing incoming calls to the appropriate person within the organization, typically to offload this task from a receptionist and also offer a simple menu system ("for menu, press 1, for transfer to phone, press 2," etc.). For more information, access the General Doc

This document covers the API(s) that are available to CP, thus helping to manage Business Customers, Auto Attendant, Phone Numbers, Extension and Users while keeping track of usage. CP also have access to API(s) exposed to Business Customers. Please sign in to your Auto Attendant Console to see the API(s) available to Business Customers.

Where ever API returns Timestamp or expects Date/Timestamp as request parameter, format is given as an ISO-8601 date/time string, like YYYY-MM-DDTHH:MM:SS[hh][mm] (for example 2019-10-10T13:13:48+0000). Note that the given date/time is inclusive and is assumed to be in UTC timezone hence offset [hh][mm] will always be +0000.

Auto Attendant API

Restcomm Auto Attendant API can be broadly classified into below categories:

  • Enterprise

  • User

  • Auto Attendant System

  • Phone Number

  • Announcement

  • Menu

  • Schedule

  • Usage Records

  • Single Sign On

Business Customer (Enterprise)

CP serve Enterprise Customers. Through their own Accounts, CP can create, manage, or delete Enterprise Customer Sub Accounts. By setting up Sub Accounts for Enterprise Customers, the CP can easily manage Auto Attendant and invoices for a specific Enterprise Customer.

Create Enterprise

This endpoint is used to create an Enterprise.

Base Resource URI

https://your_custom_domain.com

Add Enterprise Resource URI

/autoattendant/enterprise

Supported Operations

HTTP POST: Add Enterprise

Request Parameters

Parameter Description

name

Mandatory name of an enterprise. Name must be unique within the given CSP organisation.

enterpriseAdmin

Enterprise Admin (BG-ADMIN) data explained in the table below that will be assigned as the admin of this enterprise.

A Restcomm sub-account will be created with these values.

extFromPhoneNumber

Obsolete, see configuration parameter below.

Optional.

Define Calling Line Identity Presentation (CLIP) Phone Number that will be used to dial extension numbers.

If not defined, the number will be the same as the caller number which is trying to reach an extension.

configuration

Optional. Represents the configuration for this Enterprise as explained below.

extFromPhoneNumber: Same as extFromPhoneNumber defined above. The above explicit parameter will be disabled soon and only configuration will be supported.

ringBackTone: The Announcement SID assigned to this Enterprise as default Ringback Tone. Whenever a caller is waiting for a call to get connected to an Extension or external Phone Number, they will hear this Ringback Tone. The Ringback Tone can be assigned only after the Enterprise is created. Hence, this parameter takes effect only after updating the Enterprise as explained in the Update Enterprise API section below. This parameter will be ignored if passed in the Create Enterprise API.

NOTE: Business Customer can overwrite this while creating a Menu in the AA System.

The configuration should be passed as a JSON string body similar to:

"configuration": "{\"autoAttendant\": {\"extFromPhoneNumber\": \"110\",\ringBackTone\:\AN42b3d3e952ff47ecba9d487095a2bc4e\}}"

Get list of Enterprise

List Enterprise API allows users to list all the Enterprises created under that Organisation, which means under that Communications Provider account.

Base Resource URI

https://your_custom_domain.com

Get List of Enterprise Resource URI

/autoattendant/enterprise/search

Supported Operations

HTTP GET: List Enterprise

Paging

The following paging parameters are supported.

Query Parameter Description

page

Which page of Enterprise records to return, starting from 0.

pageSize

Number of records returned per page
If search API is called without page and pageSize parameters, by default pageSize will be set to 10, and page will be 0 to avoid retrieving all data in a single API call.

Filtering

The following filtering parameters are supported.

Query Parameter Description

Name

Optional parameter. Only show Enterprises that match name text, partially or fully.

startTime

Optional parameter. Only show enterprises that were created on this date/time or later, given as an ISO-8601 date/time string, like YYYY-MM-DDTHH:MM:SS (for example 2018-10-05T22:45:32) or, if you want to omit the time, YYYY-MM-DD (for example 2018-10-05). When only a date is provided the time is assumed to be at midnight of the given date. Note that the given date/time is inclusive and is assumed to be in UTC timezone.

endTime

Optional parameter. Only show enterprises that were created on this date/time or earlier, given as an ISO-8601 date/time string, like YYYY-MM-DDTHH:MM:SS (for example 2018-10-06T02:10:03) or, if you want to omit the time, YYYY-MM-DD (for example 2018-10-06). When only a date is provided the time is assumed to be at midnight of the given date. Note that the given date/time is inclusive and is assumed to be in UTC timezone.

Sorting Information

You can use the SortBy GET query parameter to determine which attribute you want to sort by and in what order. 'asc' stands for ascending and 'desc' for descending sort ordering. Heres the overall format: SortBy=<sorting attribute>:<direction>. If no direction parameter is provided, then the listing of calls is sorted by the attribute in ascending order. Below you can find the possible attributes you can sort by:

Parameter Description

User

Users can have assigned any of the roles defined below.

  • Communications Service Provider Admin (also known as CSP_ADMIN) can create Enterprise, Auto Attendant System, Phone Number, Extensions and users with role CSP_ADMIN, BG_ADMIN and AA_ADMIN. Users with roles BG_ADMIN and AA_ADMIN are used for Enterprise.

  • Enterprise Admin (also known as BG_ADMIN) can create users with roles Enterprise Admin and AA Admin (also known as AA_ADMIN).

  • Auto Attendant Admin (also known as AA_ADMIN). The only difference between Enterprise Admin and AA Admin is that the AA Admin cannot add any new users and cannot make changes to Extensions.

Usage Records

Usage Records

Numbers are either main Phone Numbers or Extension Phone Numbers added for a given Enterprise. Usage Records keep track of number additions for a given month. For example, if a number was added on the 5th day of month and removed on 25th day of the same month, it will still be counted under the Usage Records for that particular month. However, from the next month onward it will not be counted as it was deleted. If the number was not deleted, the Usage Records count will include it for next month and hence its monthly recurring count.

Base Resource URI

https://your_custom_domain.com

Get Mapping Usage Resource URI

/autoattendant/phonenumber/usage/records

Supported Operations

HTTP GET: List usage records for numbers

Filtering

Usage filters will return usage records for the passed filter. The following filtering parameters are supported.

Query Parameter Description

startTime

Only include usage that has occurred on or after this date. The format is YYYY-MM-DD. All dates are in GMT. As a convenience, you can also specify an offset to today. For example, StartDate=-30days will make the StartDate be 30 days before today.

endTime

Only include usage that has occurred on or before this date. The format is YYYY-MM-DD. All dates are in GMT. As a convenience, you can also specify an offset to today. For example, EndDate=+30days will make the EndDate be 30 days from today.

enterpriseSid

Optional. Only include Usage records for specific Enterprise.

If the CSP_ADMIN calls this API and without passing the enterpriseSid parameter, the Usage Records for the entire Organisation will be retrieved.

This parameter is ignored for ENTERPRISE_ADMIN users.
