Campaign Manager

Introduction

Campaign Manager allows Communications Providers to deliver SMS and Voice campaign management solutions to their Business Customers, as well as manage users with different roles and capabilities. Business Customers can schedule the same SMS or Voice calls to hundreds of thousands of phone numbers. For more information, please check out the Campaign Manager General Documentation.

This document covers the API(s) available for use by the Communications Providers, that can help them manage Business Customers, while keeping track of their usage.

Business Customers

Communications Providers manage Business Customers (also known as Enterprises). They can create, update, read, and delete Business Customer accounts. The logical grouping of Business Customer accounts under a Communications Provider helps in managing the billing and invoices for the Business Customers.

Business Customer Status

At any given point in time, Business Customers can have one of the following statuses:

  • ACTIVE: Can use Campaign Manager.

  • INACTIVE: The Business Customer is marked as inactive:

    • When all Business Customer users are marked as INACTIVE, the status change is also reflected in the CPaaS Platform.

    • Campaigns that are in Scheduled state will be marked as Failed.

    • Campaigns that are InProgress will be stopped and marked as Failed. Any SMS messages that are yet to be delivered will not be sent.

    • If the status is changed from INACTIVE to ACTIVE, all the users in Campaign Manager and CPaaS will be moved back to ACTIVE.

      • No change in Campaigns should happen.

  • CLOSED: Once set to CLOSED, the status cannot be changed to INACTIVE or ACTIVE.

    • Business Customer users are CLOSED in the CPaaS Platform as well.

    • Campaigns that are in Scheduled state will be marked as Failed.

    • Campaigns that are InProgress will be stopped and marked as Failed. If there are any SMS messages that are yet to be delivered will also fail.

Once the status is set to CLOSED, all resources associated with the Business Customer will be deleted. However, as of today resources will still be kept in the database as per the CPaaS platform retention settings.

Create Business Customer

This endpoint is used to create a Business Customer in the Campaign Manager application. Create Business Customer will add unique Business Customer names for a given Communications Provider.

Base Resource URI

https://your_custom_domain.com

Create Business Customer Resource URI

/campaignmanager/enterprises

Supported Operations

HTTP POST: Create Business Customer

Request Parameters

Parameter Description

name

Mandatory. Name of the Business Customer. Name must be unique within a given Communications Providers Organisation.

configuration

Optional. Represents throttling for SMS and Voice assigned to the Business Customer. If not passed, it defaults to using maximum throttling available at a given point of time.

Configuration should be passed as a JSON String as shown below:

\{\"campaignManager\":\{\"throttle\": \{\"smsPerSecond\":\"10\",
\"callsPerSecond\":\"25\", \"concurrentCalls\":\"5\"}}}

  • smsPerSrcond: SMS per second throttling applied to this Business Customer

  • callsPerSecond: Call per second throttling applied to this Business Customer

  • concurrentCalls: Number of concurrent calls allowed at a given point in time.

If the value passed is 0 or any negative number, the request will fail.

status

Optional. Status of the Business Customer. Default value is ACTIVE. Possible values are ACTIVE and INACTIVE.

enterpriseAdmin

Mandatory. Business Customer user data explained in the table below that will be assigned as the admin of this business customer.

A corresponding subaccount will be created in CPaaS with these values.

The response body will carry an error code, a corresponding message, and data of the Business Customer if created successfully. The HTTP response will be 200 Created (OK).

Get List of Business Customers

This endpoint is used to search a list of Business Customers.

Base Resource URI

https://your_custom_domain.com

Get List of Business Customer Resource URI

/campaignmanager/enterprises/search

Supported Operations

HTTP GET: Get Business Customer

Paging

The following paging parameters are supported

Query Parameter Description

page

Which page of Business Customer records to return, starting from 0.

pageSize

Number of records returned per page
If search API is called without page and pageSize, by default pageSize will be 10 and page will be 0 to avoid retrieving all data in a single API call.
Filtering

Following filtering parameters are supported:

Query Parameter Description

name

Optional parameter. Only show Business Customers that match the name text, partially or fully.

Status

Optional parameter. Only show Business Customers that have this particular status.

startTime

Optional parameter. Only show Business Customers that were created on this date/time or later, given as an ISO-8601 date/time string, like YYYY-MM-DDTHH:MM:SS (for example 2018-10-05T22:45:32) or, if you want to omit the time, YYYY-MM-DD (for example 2018-10-05). When only a date is provided the time is assumed to be at midnight of the given date. Note: that the given date/time is inclusive and is assumed to be in UTC timezone.

endTime

Optional parameter. Only show Business Customers that were created on this date/time or earlier, given as an ISO-8601 date/time string, like YYYY-MM-DDTHH:MM:SS (for example 2018-10-06T02:10:03) or, if you want to omit the time, YYYY-MM-DD (for example 2018-10-06). When only a date is provided the time is assumed to be at midnight of the given date. Note that the given date/time is inclusive and is assumed to be in UTC timezone.
Sorting Information

You can use the sortBy GET query parameter to determine which attribute you want to sort by and in which direction; direction can either be 'asc' for ascending and 'desc' for descending sort ordering. Heres the overall format: sortBy=<sorting attribute>:<direction>. If no direction parameter is provided, then the listing of calls is sorted by the attribute in ascending order. You can sort by the attributes listed below:

Parameter Description

name

Sort by name

dateCreated

Sort by created date

Users

User Role and Status

Communications Providers can retrieve user information and search user records.

Communications Providers and Business Customers are limited to having a single User per each entity.

A User can have any of the following roles assigned to them.

  • CP_ADMINISTRATOR:

    • CRUD Business Customers (including credit and throttle)

    • CRUD CP_ADMINISTRATOR

    • CRUD BC_TURNKEY_ADMINISTRATOR

    • CRUD CP_PROVISIONING_AGENT

    • Read CDR and Usage Records for Business Customers (**)

  • CP_PROVISIONING_AGENT:

    • CRUD Business Customers (including credit and throttle)

    • CRUD BC_TURNKEY_ADMINISTRATOR

    • Read CDR and Usage Records for Business Customers (**)

  • BC_TURNKEY_ADMINISTARTOR:

    • CRUD campaign

    • CRUD BC_TURNKEY_ADMINISTRATOR

Campaign

Campaign Status

A campaign can have any of the following statuses:

  • Initiating: The Campaign is getting created.For example, setting up From numbers, To numbers, and Body.The Campaign will remain in this state for a very short time.If there are any errors in processing the From or To numbers, the Campaign will directly move to a Failed state.

  • Scheduled: The Campaign is Scheduled.This means the Campaign Manager has all the resources and data it needs to execute this Campaign when the scheduled time kicks in.You can transition a Campaign from a Scheduled to a Cancelled state is via an API call.

  • Cancelled: When a scheduled Campaign is cancelled.

  • InProgress: The Campaign is in progress. This state automatically kicks in when scheduled time has arrived. The only possible transition from this state is either Completed or Failed

  • Completed: Once all SMS are successfully sent to the underlying platform, the Campaign is marked as completed.It doesnt depend on the state of SMS, for example whether it is Delivered or not.

  • Failed: If the Campaign failed for some reason or cannot setup data correctly from the initiating state.

Estimate the Volume of the Campaign

This endpoint is used to estimate the volume of a given campaign. The following section outlines the process of estimating the campaign’s volume.

  • If you have provided an OptOut list, the same will be retrieved from the system.

  • If you have provided an OptIn list, the same will be retrieved from the system, and then the items from the OptIn list will be excluded from the OptOut List.

  • When the To List is retrieved from the system, the items from the OptOut list are excluded from the To List.

  • The final Recipients List corresponds to the following equation: \(Recipients List = ToList - (OptOut - OptIn)\)

  • The Body section of the campaign is checked to see if it contains template variables (like "Hello ${name}").

  • Each recipient line is re-formatted to replace parameters from the columns in the CSV file.

  • When each line has its text finalized, the number of SMS segments for that line is calculated taking into account the encoding of the text (SMS Encoding).

  • Finally, the sum of all lines is returned to the user.

Numbers Validation

The phone numbers validation happens on two levels: Basic validation and CSV Validation.

Basic validation

The following section outlines the aspects of basic phone numbers validation.

  • The file containing the phone numbers must not be empty.

  • The file containing the phone numbers shall not be bigger than 100 MB or shall not contain more than 1,000,000 lines.

  • The recipients' numbers may contain only digits, + and signs.

  • The + sign can only be the first character of the line.

  • Surplus spaces at the beginning or end of the line will be trimmed.

  • Any sign will be trimmed and replaced with an empty string.

  • Any space in the middle of a phone number will be trimmed and replaced with an empty string.

  • Empty lines will be trimmed.

  • If the file contains empty entries, the same will be marked as invalid.

  • The To and DND Phone numbers must contain minimum 7 characters.

  • Phone numbers must contain maximum 15 characters.

CSV Validation

The following section outlines the aspects of CSV file validation.

  • The CSV file must contain a header line.

  • The header line must contain a number column.

  • Each header cells name will be trimmed, if spaces are found in the beginning or end of its value.

  • Each CSV line must contain the same number of columns as the header line does

  • Each CSV row will be limited to 255 characters.

  • A cell value will be trimmed if spaces are found in the beginning or end of it.

  • The phone number in the number column is validated via Basic validation.
