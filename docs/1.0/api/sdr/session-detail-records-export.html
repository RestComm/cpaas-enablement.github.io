Session Detail Records Export

Overview

Session Detail Records (SDR) represent the Voice and Messaging sessions generated by an organization or a user.

The SDR can be exported through the SDR Export functionality that exposes an SFTP server allowing a quick and easy download of the SDR records. Search queries per SDR type are supported through the corresponding APIs.

This feature is designed for Communication Providers to aid the process of monitoring and billing the traffic of their Business Customers.

Periodical SDR exports ensure easier billing process and analysis of data.

File Format

The SDR files are compressed using GZip to reduce bandwidth and speed up downloading. There is a single CSV file inside the compressed file.

The first line of the CSV file contains the names of the columns included. Any application built upon such CSV file needs to take into consideration its column names.

New columns may be added at any time as the SDR format evolves

The order of the columns is not guaranteed and may vary at any time

Depending on profile settings or even the content itself, some column names or field contents may be wrapped in double quotes

Export Frequency

The CPaaS platform uploads a new CSV file per SDR type on an hourly basis.

Each CSV file contains the SDRs for the traffic sessions that were completed the previous hour.

Note that a given SDR record will be included in the hour interval corresponding to when the SDR record reached its final state and not when it was initially created.

If no traffic was generated during a given hour, the corresponding CSV file will still be created with no data inside its rows.

Directory Convention

The files are uploaded following a time rotation convention. This convention allows generating files with predictable names, so simple scripts can be created to download all content without requiring listing files.

The convention follows the following pattern <SDR-Type>/yyyy/mm/dd/<SDR-Type>-yyyy-mm-dd-hh.gzip:

  • "yyyy" Is the year

  • "mm" Is the month of the year ranging from "01" to "12".

  • "dd" Is the day of the month ranging from "01" to "31" (depending on the number of days in the month)

  • "hh" Is the hour of the day ranging from "00" to "23"

SDR Types

The CSV column names are based on the flattened JSON representation of SDR, where nested fields are separated by the . (dot) character.

The following SDR types are currently supported:

  • Call

  • Conference

  • Gather DTMF

  • Gather Speech

  • Hold

  • Play

  • Recording

  • Refer

  • Resume

  • Say

  • Message

SDR Schema

An OpenAPI schema has been provided to describe each field of the different SDR type. You can find the OpenAPI schema here https://app.swaggerhub.com/apis/Telestax/restcomm-accounting_api/1.1.5

Authentication

The SFTP username must be set to the regular Account Sid. The SFTP password must be set to the Account token.

The associated organization to the authenticated Administrator account will be used to determine the corresponding SDR files in the SFTP session.

All the files exposed are read only and writing or removal is explicitly forbidden.

Retention Policy

The SDR Export functionality automatically removes files older than six months.

SDR Provisioning Settings and Enablement

In order to enable SDRs Export for your Organization, please contact your account manager.

CSV Settings

Depending on which billing system you use, you may need to change the default CSV separator and columns to be included. Any of the column names available in the CSV file can be included/excluded from the export.

curl -X PATCH https://yourcompany.com/sps/private \
  -H 'Content-Type: application/merge-patch+json' \
  -u 'YourAccountSid:YourAuthToken' \
  -d '{
  "data": {
    "sdr_export_settings": {
      "sdr_types": {
        "calls": {
          "include": "date_created,date_updated,from,to"
        },
        "refers": {
          "exclude": "sid,org_sid"
        }
      },
      "enabled": true,
      "separator": ","
    }
  }
}'

Usage

You can retrieve Calls SDRs by running the following query in your terminal.

Make sure you make a request to the Point of Presence your organization is associated with.

Point of Presence Domain Name

USA

ftp.cloud.restcomm.com

Europe

ftp.eu.restcomm.com

Japan

ftp.jp.restcomm.com

India

ftp.in.restcomm.com

 
sftp <accountsid>@ftp.cloud.restcomm.com:/calls/2021/01/01/calls-2021-01-01-00.gzip/some/local/folder

When prompted for a password enter the corresponding Account Token. This example is based on the Linux sftp tool.
