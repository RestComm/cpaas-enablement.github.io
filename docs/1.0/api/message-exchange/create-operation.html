Create Operation

Create allows Communication Service Providers (CSP) Account (Restcomm Account) to create MX mappings and new Enterprise names for easy segregation of mappings specific to each enterprise account. For example consider Acme as CSP will have Kohls, Walmart etc as Enterprise Customers. Acme should be able to define the Enterprise and within each Enterprise add unique MX mappings

Create Enterprise

Create Enterprise will add unique Enterprise names for given CSP.

Base Resource URI

https://<custom domain of CSPs>

Add Enterprise Resource URI

/messageexchange/enterprise

Supported Operations

HTTP POST: Add Enterprise

Request Parameters

Parameter Description

name

Mandatory name of Enterprise. Name must be unique within given CSP Organisation.

Response Body

Response Body will carry Code, corresponding message and data of Enterprise if it was successfully created. HTTP response will be 200 OK.

The response returned is JSON body as shown in the example below. Each parameters are as explained below

Parameter Description

code

Sub-error code in case of error

message

Message describing error if error occured or success message

data

Enterprise data as explained in the table below.

Enterprise Data Parameter

Data Parameter Description

sid

Unique Enterprise Id

accountSid

Unique Restcomm Account that created/updated this particular Enterprise

accountEmail

Email associated with accountSid

dateCreated

Timestamp when this Enterprise was created.

dateUpdated

Timestamp when this Enterprise was updated

name

Name of this Enterprise

mappingsCount

Number of Jabber Address to SMS enabled Numbers mappings under this Enterprise

uri

Unique URI that you can call to get only this specific Enterprise

Create Enterprise API can also return Error Code and corresponding HTTP Error as mentioned in Common Response Error Code. In addition below are the Error that it can return

Error Code

HTTP Status

Error Message

Description

Category

460

409

Enterprise name not unique

Add Enterprise API is called for enterprise name that is already present for this Organisation. There can be same name Organisation in different Orgs.

Create Enterprise, Update Enterprise

Example:

{
 "name" : "walmart"
}

From the bash terminal you can run the command below:

curl -X POST \
  https://cloud.restcomm.com/messageexchange/enterprise \
--user  ' {your_account_SID}:{your_account_token}' \
  -H 'Cache-Control: no-cache' \
  -H 'Content-Type: application/json' \
  -d  {
 "name" : "walmart"
}

If creation of Enterprise is successful, below is the response returned:

{
    "data": {
        "sid": "EN263d95d2bafa4e4a9e6a5563851cd283",
        "dateUpdated": "2019-10-11T11:11:00+0000",
        "dateCreated": "2019-10-11T11:11:00+0000",
        "accountSid": "ACa91f3a070d55122d9ec0069b0be27758",
        "accountEmail": "subhahu.reddy@telestax.com",
        "uri": "/messageexchange/enterprise/search/EN263d95d2bafa4e4a9e6a5563851cd283",
        "name": "walmart",
        "mappingsCount": 0
    },
    "code": 200,
    "message": "OK"
}

If creation fails, below response is returned with corresponding HTTP Error Code

{
    "errorCode": "460",
    "message": "Enterprise name not unique",
    "data": {
        "name": "walmart"
    }
}

Create MX Mappings

Create MX Mapping will add mapping between Jabber Address and SMS enabled number. === Base Resource URI https://<custom domain of CSPs>

Add MX Mappings Resource URI

/messageexchange/mappings

Supported Operations

HTTP POST: Add MX users and SMS enabled numbers mappings

Request Parameters

Parameter

Description

name

Optional friendly name of MX user. If not passed its same as MX UserId

jabberAddress

Mandatory MX User ID to be mapped. This must be unique in whole system

externalAddress

Mandatory number that must be mapped with userId. System will also SMS enable this number. This must be unique in the whole system.

enterpriseSid

Mandatory Enterprise id that was added via Create Enterprise API call.

domainSid

Optional parameter. User has to first create Domain as explained in section 2.3 Create Domain and pass the Domain Sid here to setup custom domain. If not passed this is set to null which means restcomm.com domain will be used.

Create MX Mapping will use the domain name created in section 2.3 Create Domain. If not defined the default domain which is restcomm.com will be used.

Response Body

Response Body will carry Code, corresponding message and data of Mapping if it was successfully created. HTTP response will be 200 OK.

Parameter

Description

code

Sub-error code for response

message

Message describing error if error occured or success message

data

Mapping data as explained in the table below.

Mapping Data Parameter

Data Parameter

Description

sid

Unique MX mapping Id

enterpriseSid

Enterprise Id under which this Mapping is created

enterpriseName

Name of Enterprise under which this Mapping is created

accountSid

Unique Restcomm Account that created/updated this particular Enterprise

accountEmail

Email associated with accountSid

dateCreated

Timestamp when this Mapping was created

dateUpdated

Timestamp when this Mapping was updated

name

Friendly name of this user

jabberAddress

Jabber Address id of this user

externalAddress

SMS enabled number mapped to this WebEx id.

domainSid

Domain Sid that is setup for this mapping.

domain

Domain name that is setup for this mapping.

uri

Unique URI that you can call to get only this specific mapping SDR

Response Parameters

Create Domain

Domain allows CSP to use custom domain that will be appended to Mobile number while delivering the message to UC One Client. If domain name is not setup, by default the receiver of message would see message coming from xxx@restcomm.com where xxx is mobile number of message sender.

CSP can setup multiple domains and have mappings using either of created domains.

Base Resource URI

https://<custom domain of CSPs>

Add Domain Resource URI

/messageexchange/domains

Supported Operations

HTTP POST: Add domain

Request Parameters

Parameter

Description

name

Name of domain. For example text.acme.com

certificate

Full certificate of domain. See further details at https://www.restcomm.com/docs/message-exchange.html#_use_your_custom_domain

Response Body

Response Body will carry Code, corresponding message and data of Domain if it was successfully created. HTTP response will be 200 OK.

Parameter

Description

code

Sub-error code for response

message

Message describing error if error occured or success message

datasid

Domain data as explained in the table below.

Domain Data Parameter

Data Parameter

Description

sid

Unique Domain Id

accountSid

Unique Restcomm Account that created/updated this particular Domain

accountEmail

Email associated with accountSid

dateCreated

Timestamp when this Domain was created

dateUpdated

Timestamp when this Domain was updated

name

Domain name

certificate

SSL Certificate setup for this domain

mappingsCount

Number of mappings using this domain

uri

Unique URI that you can call to get only this specific mapping SDR

Response Parameters

Create IP Whitelisting

Whitelisting IP allows the messages from whitelisted IPs only. The IP should be of peer XMPP server.

Base Resource URI

https://<custom domain of CSPs>

Add WhiteList IP Resource URI

/messageexchange/whitelist

Supported Operations

HTTP POST: Add IP Whitelisting

Request Parameters

Parameter

Description

ipAddress

IP Address to be whitelisted in IPv4 format only.

Response Body

Parameter
