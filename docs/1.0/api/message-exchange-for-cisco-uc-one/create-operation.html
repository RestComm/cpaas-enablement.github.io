Create Operation

Create allows Communication Providers Account (Restcomm Account) to create message exchange mappings and new Enterprise names for easy segregation of mappings specific to each enterprise account. For example consider Acme as CSP will have Kohl’s, Walmart etc as Enterprise Customers. Acme should be able to define the Enterprise and within each Enterprise add unique MX mappings

Create Enterprise

Create Enterprise will add unique Enterprise names for given CSP.

Base Resource URI

https://your_custom_domain.com

Add Enterprise Resource URI

/messageexchange/enterprise"]

Supported Operations

HTTP POST: Add Enterprise

Request Parameters

Parameter Description

name

Mandatory name of Enterprise. Name must be unique within given CSP Organisation.

Response Body

Response Body will carry Code, corresponding message and data of Enterprise if it was successfully created. HTTP response will be 200 OK.

Sub-error code in case of error

Name of this Enterprise

Number of Jabber Address to SMS enabled Numbers mappings under this Enterprise

Create Message Exchange Mappings

Create Message Exchange Mapping will add a mapping for a Jabber Address with SMS enabled number and provision the number with the network carrier.

Lifecycle of number mapping

Calling Create Message Exchange Mapping endpoint initiates SMS enablement of a number, provisioning it in Restcomm and creating a corresponding mapping in Message Exchange. This process is asynchronous. The diagram below shows various states of mapping and possible transitions from one state to another.

As soon as create mapping request is sent, Message Exchange (MX) will immediately send back HTTP 200 OK Response with a “PROCESSING” status of the mapping.

There are two possible outcomes. Either the number gets successfully provisioned by the carrier or it fails.

  • If the number is successfully SMS enabled, the system will provision it in Restcomm by either adding a new number or updating an existing number. The status of the mapping will be updated to “SUCCESS”. If there is a failure from the Restcomm side the “deleteFailReason” field will be updated with the corresponding error message for that mapping.

  • If the SMS enabling of a number failed, the mapping status will be updated to “FAILED” and the respective failed reason will be stored for that order in the deleteFailReason field. The number won’t be provisioned in Restcomm. Create mapping endpoint can be called again.

Number provisioning is asynchronous process. The user can call the create mapping API multiple times for the same number and jabber address until a successful order is made. In that case all other provisioning requests with same number will be marked as failed.

Base Resource URI

https://your_custom_domain.com

Add MX Mappings Resource URI

/messageexchange/mappings

Supported Operations

HTTP POST: Add MX users and SMS enabled numbers mappings

Request Parameters

Parameter

Description

name

Optional friendly name of MX user. If not passed its same as MX UserId

jabberAddress

Mandatory MX User ID to be mapped. This must be unique in whole system

externalAddress

Mandatory number that must be mapped with userId. System will also SMS enable this number. This must be unique in the whole system.

enterpriseSid

Mandatory Enterprise id that was added via “Create Enterprise” API call.

domainSid

Optional parameter. User has to first create Domain as explained in section 2.3 Create Domain and pass the Domain Sid here to setup custom domain. If not passed this is set to null which means “restcomm.com” domain will be used.

Create MX Mapping will use the domain name created in section 2.3 Create Domain. If not defined the default domain which is restcomm.com will be used.

Response Body

Response Body will carry Code, corresponding message and data of Mapping if it was successfully created. HTTP response will be 200 OK.

Create Domain

Domain allows CSP to use custom domain that will be appended to Mobile number while delivering the message to UC One Client. If domain name is not setup, by default the receiver of message would see message coming from xxx@restcomm.com where xxx is mobile number of message sender.

CSP can setup multiple domains and have mappings using either of created domains.

Base Resource URI

https://your_custom_domain.com

Add Domain Resource URI

/messageexchange/domains

Supported Operations

HTTP POST: Add domain

Request Parameters

Parameter

Description

name

Name of domain. For example “text.acme.com”

certificate

Full certificate of domain. See further details at https://www.restcomm.com/docs/message-exchange.html#_use_your_custom_domain

Response Body

Response Body will carry Code, corresponding message and data of Domain if it was successfully created. HTTP response will be 200 OK.

Parameter

Description

Sub-error code for response

message

Message describing error if error occured or success message

dateUpdated

name

SSL Certificate setup for this domain
