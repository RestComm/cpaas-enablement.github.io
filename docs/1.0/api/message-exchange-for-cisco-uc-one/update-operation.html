Update Operation

Update Enterprise

Update Enterprise allows to update existing Enterprise.

Base Resource URI

https://your_custom_domain.com

Update Enterprise Resource URI

/messageexchange/enterprise/{EnterpriseId}

Supported Operations

HTTP PUT: Update Enterprise

Request Parameters

Parameter

Description

EnterpriseId

Enterprise SID that was generated by calling create Enterprise API

name

Mandatory name of Enterprise. Name must be unique within given CSP Organisation.

Response Body

Response Body will carry Code, corresponding message and data of Enterprise if it was successfully Updated. HTTP response will be 200 OK.

Parameter

Description

code

Sub-error code for response

message

Message describing error if error occured or success message

data

Enterprise data as explained in Enterprise Data Parameter.

Response Parameters

Create Enterprise API will return Error Code and corresponding HTTP Error as mentioned in Common Response Error Code. In addition below are the Error that it can return

Error Code

HTTP Status

Error Message

Description

Category

460

409

Enterprise name not unique

Update Enterprise API is called for enterprise name that is already present for this Organisation. There can be same name Organisation in different Orgs.

Create Enterprise, Update Enterprise

467

409

Enterprise not found

Enterprise which is to be updated is not found.

Create Mapping, Update Mapping, Update Enterprise

Example

{
    "name" : "Kohl"
}

From the bash terminal you can run the command below:

curl -X PUT \
  https://cloud.restcomm.com/messageexchange/enterprise/MXEb6eb071d21124dbab20b095c25f1274f \
--user  ' \{your_account_SID}:\{your_account_token}' \
  -H 'Cache-Control: no-cache' \
  -H 'Content-Type: application/json' \
  -d '{
    "name" : "Kohl"
}

If update of Enterprise is successful, below is the response returned:

{
    "data": {
        "sid": "EN263d95d2bafa4e4a9e6a5563851cd283",
        "dateUpdated": "2019-10-11T11:12:23+0000",
        "dateCreated": "2019-10-11T11:11:01+0000",
        "accountSid": "ACa91f3a070d55122d9ec0069b0be27758",
        "accountEmail": "jane.doe@company.com",
        "uri": "/messageexchange/enterprise/search/EN263d95d2bafa4e4a9e6a5563851cd283",
        "name": "Kohl",
        "mappingsCount": 0
    },
    "code": 200,
    "message": "OK"
}

If update of Enterprise fails, corresponding error code and message is returned along with HTTP Error Response

{
    "errorCode": "460",
    "message": "Enterprise name not unique",
    "data": {
        "name": "Kohl"
    }
}

Update MX Mappings

Update MX Mapping will update mapping between JabberAddress (UC One / XMPP Address) and SMS enabled number.

Base Resource URI

https://your_custom_domain.com

Update MX Mappings Resource URI

/messageexchange/mappings/{mappingId}

Supported Operations

HTTP PUT: Update mappings.

Note: jabberAddress and externalAddress cannot be updated. If you need to change any of these, you have to delete the existing mapping and recreate a new one.

Request Parameters

Parameter

Description

mappingId

Mandatory unique Jabber Address vs SMS enabled number mapping id that was generated by calling Add Mapping API

name

Optional friendly name of MX user. If not passed its same as jabberAddress record

enterpriseId

Optional Enterprise id. If user wants to move Mapping from one enterprise to another, pass the enterpriseId of target Enterprise. If not passed, use same as set in original mapping.

domainSid

Optional parameter. If user wants to move Mappings from one domain to another, pass the Domain Sid here to change custom domain. If not passed, use same as set in original mapping.

Response Body

Response Body will carry Code, corresponding message and data of Mapping if it was successfully Updated, HTTP response will be 200 OK.

Code

Description

code

Sub-error code for response

message

Message describing error if error occured or success message

data

Mappings data as explained in Mapping Data Parameter

Response Parameters

Update MX mapping API will return Error Code and corresponding HTTP Error as mentioned in Common Response Error Code

Error Code

HTTP Status

Error Message

Description

Category

467

409

Enterprise not found

Enterprise for which mapping is to be added is not found.

Create Mapping, Update Mapping, Update Enterprise

409

Mapping not found

Update is called passing MappingId that is not found for give Organisation

Update Mapping

474

409

Domain Id not found

Domain is not found.

  -H 'Content-Type: application/json' \

}

If update of mapping is successful, below is the response returned:

}

Update Domain

Updating domain to update domain name or certificate

Base Resource URI

https://your_custom_domain.com

Update Domain Resource URI

/messageexchange/domains/{domainId}

Supported Operations

HTTP PUT: Update Domain

Request Parameters

Parameter

Description

domainId

Unique Id generated while creating the domain

name

Name of domain. For example text.acme.com

certificate

Full certificate of domain. See further details at https://www.restcomm.com/docs/message-exchange.html#_use_your_custom_domain

Response Body

Response Body will carry Code, corresponding message and data of Domain if it was successfully Updated, HTTP response will be 200 OK.

Code

Description

code

Sub-error code for response

message

Message describing error if error occured or success message

data

domain data as explained in Domain Data Parameter

Response Parameters

Update Domain API will return Error Code and corresponding HTTP Error as mentioned in Common Response Error Code. In addition below are the Error that it can return

Error Code

HTTP Status

Error Message

Description

Category

Update IP Whitelisting

Updating whitelisting allows CSP to change the IP whitelisted.

Base Resource URI

https://your_custom_domain.com

Update IP Whitelisting Resource URI

/messageexchange/whitelist/{whitelistId}

Supported Operations

HTTP PUT: Update IP Whitelist

Request Parameters

Parameter

Description

whitelistid

Unique Id generated while creation of Whitelist Ip

ipAddress

IP Address to be whitelisted in IPv4 format only.

Response Body

Response Body will carry Code, corresponding message and data of Whitelist IP if it was successfully Updated, HTTP response will be 200 OK.

Code

Description

code

Sub-error code for response

message

Message describing error if error occured or success message

data

Whitelist IP data as explained in IP Whitelist Data Parameter

Response Parameters

Update Whitelist IP API will return Error Code and corresponding HTTP Error as mentioned in Common Response Error Code. In addition below are the Error that it can return

Error Code

HTTP Status

Error Message

Description

Category

475

400

Whitelist IP Id not found

Update Whitelist IP API is called that is not present in system

Update Whitelist IP, Delete Whitelist IP
