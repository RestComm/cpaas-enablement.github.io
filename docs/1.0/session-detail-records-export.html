Session Detail Records (SDRs) Export

Overview

Session Detail Records (SDRs) represent the Voice and Messaging sessions generated by an organization or a user. This feature is designed for Communication Providers to aid the process of monitoring and billing the traffic of their Business Customers. Periodical SDR exports ensure easier billing process and analysis of data.

SDRs can be exported through the SDR Export functionality that exposes an SFTP server allowing a quick and easy download of the SDR records. Search queries per SDR type are supported through the corresponding APIs.

File Format

SDR files are compressed using GZip to reduce bandwidth and speed up downloading. There is a single CSV file inside the compressed file. The first line of it contains the names of columns included. New columns can be added at any time. Any application built upon such CSV file needs to take into consideration its column names.

Export Frequency

The CPaaS platform is uploading a new CSV file per SDR type hourly. Each CSV file contains the SDRs for the traffic sessions that were completed the previous hour. Note that a given SDR record will be included in the hour interval corresponding to when the SDR record reached its final state and not when it was initially created. If no traffic was generated during a given hour, the corresponding CSV file will still be created with no data inside its rows.

Directory Convention

The files are uploaded following a time rotation convention. This convention allows generating files with predictable names, so simple scripts can be created to download all content without requiring listing files. The convention follows the following pattern "<SDR-Type>/yyyy/mm/dd/<SDR-Type>-yyyy-mm-dd-hh.gzip":

  • "yyyy" Is the year

  • "mm" Is the month of the year ranging from "01" to "12".

  • "dd" Is the day of the month ranging from "01" to "31" (depending on the number of days in the month)

  • "hh" Is the hour of the day ranging from "00" to "23"

Authentication

The SFTP username must be set to the regular Account Sid. The SFTP password must be set to the Account token. The associated organization to the authenticated Administrator account will be used to determine the corresponding SDR files in the SFTP session. All the files exposed are read only and writing or removal is explicitly forbidden.

Retention Policy

The SDR Export functionality automatically removes files older than six months.

SDR Types

The Type columns supported are based on the corresponding representation at the API level. The CSV column names are based on the corresponding JSON representation. Nested fields are separated by "." character.

Supported SDR types and corresponding APIs

  • "calls" Calls API

  • "conferences" Conferences API

  • "gathers_dtmf" Gather DTMF API

  • "gathers_speech" Gather Speech API

  • "holds" Hold API

  • "pauses" Pause API

  • "plays" Play API

  • "recordings" Recording API

  • "refers" Refers API

  • "resumes" Resume API

  • "says" Say API

  • "sms_messages" SMS API

SDR Provisioning Settings and Enablement

In order to enable SDRs Export for your Organization, please contact your account manager.

CSV Settings

Depending on which billing system you use, you may need to change the default CSV separator and columns to be included. Any of the column names available in the CSV file can be included/excluded from the export.

Example

curl -X PATCH https://yourcompany.com/sps/private \
  -H 'Content-Type: application/merge-patch+json' \
  -u 'YourAccountSid:YourAuthToken' \
  -d '{
  "data": {
    "sdr_export_settings": {
      "sdr_types": {
        "calls": {
          "include": "date_created,date_updated,from,to"
        },
        "refers": {
          "exclude": "sid,org_sid"
        }
      },
      "enabled": true,
      "separator": ","
    }
  }
}'

You can retrieve Calls SDRs by running the following query in your terminal. Make sure you make a request to the Point of Presence your organization is associated with.

Point of Presence Domain Name

USA

ftp.cloud.restcomm.com

Europe

ftp.eu.restcomm.com

Japan

ftp.jp.restcomm.com

India

ftp.in.restcomm.com

 
sftp <accountsid>@ftp.cloud.restcomm.com:/calls/2021/01/01/calls-2021-01-01-00.gzip /some/local/folder

When prompted for a password enter the corresponding Account Token. This example is based on the Linux sftp tool.
