Restcomm Message Exchange for Cisco Webex Teams Overview

Overview

Restcomm Cisco Webex helps service providers enable their Enterprise customers to send and receive SMS using Webex Teams and to manage and monitor their usage. Enterprise customers can send two-way SMS.

If you would like to start using the Restcomm Cisco Webex Teams feature, you need to complete the following steps:

  • Create Bot for each of the Enterprises and map the Bot Access Token to the Enterprise as explained in Create Bot for Enterprise documentation.

  • Create a mapping (via the API or the user interface of the Console)

  • Start sending messages

Prerequisite

In order to complete the aforementioned steps, you will need:

  • An activated Restcomm Cloud account.

  • Webex Teams setup for your Enterprise customers

  • Working knowledge of cURL and a way to make HTTP API calls (either using cURL or Postman; We recommend using Postman, if you’re not familiar with cURL.) Or you can directly use the interface of the Restcomm Cisco Webex Console.

  • An understanding of how to add mappings via the Restcomm API for Cisco Webex Teams or the Console.

You need to reach out to your Technical Account Manager to make sure your Account has been correctly set up for Cisco Webex Teams SMS enablement.

Setting up your first mapping

You must have a valid, working Restcomm account in order to complete this step. If you do not have one, you can contact us.

The purpose of Webex Teams mapping is to associate a phone phone number with an existing Cisco Webex Teams Id. For example, imagine you have a user with the Webex Teams id of joe@myserver.com and Restcomm number is +1-919-555-1212.

It also supports setting up Group Messaging where the number +1-919-555-1212 is mapped to multiple Webex Teams Ids, like joe@myserver.com, bob@myserver.com, etc.

Mapping will tell Restcomm for Cisco Webex Teams service to send all inbound SMS messages, sent to phone number (+1-919-555-1212), to Webex Teams Space. Where joe@myserver.com is part of, or in case of Group Messaging, both Joe and Bob are added in that space. Additionally, any time Joe (or Bob if part of the same space) sends an outbound SMS from the Cisco Webex Teams account, it will be sent from a mapped phone number.

You can set up your mapping in two different ways, either via the Restcomm Message Exchange for Cisco Webex Teams Console or via the API.

Setting up a new mapping via Console

  • Log into your Restcomm Message Exchange for Cisco Webex Teams Console via your Restcomm cloud account

  • Create a new mapping as explained in Add Mappings Console documentation.

  • Before creating a mapping, you need to create an Enterprise. This is a very important step as Cisco Webex Bot has a one-to-one mapping with the Enterprise.

  • Create Bot (for each enterprise) as explained in the Create a Bot for Enterprise documentation.

Receiving an SMS/message in Webex Teams

Once you’ve completed all of the steps above, you’re ready to test.

To begin, send a message from a PSTN number to the mapped number. You should see the message appearing in the Webex Teams space. The message should appear in a new space if you are receiving a message for the first time. All subsequent messages will appear in the same space. The default space name is set to “Phone Number:xxx”, where phone number is the sender’s number and xxx is the name provided while creating the mapping. You can rename the space to something more meaningful, for example, “Ryan’s Mobile Phone:Group 1”.

Replying to received SMS/message

To reply to received message follow the steps below:

  • In newly created space, refer to the Bot with “@” prefix and then type your message.

  • Hit enter and your message will be sent as SMS to the recepient’s mobile number.

Sending an SMS/message to new Number from Webex Teams

To send SMS to new number, follow below steps:

  • Add newly created Bot as Contact in your Cisco Webex if not done earlier. This is a one time process.

  • Send any private message to this Bot

  • It should respond back with Card that allows you to enter the receiving party’s phone number

    • If you are part of multiple mappings, it also allows you to select via which mapping you want to send out SMS.

  • Enter the recepient’s phone number and hit Submit. It should create a new space with “recepient’s phone number:Mapping name” as Space name

  • In newly created space, refer to the Bot with “@” prefix and then type your message.

  • Hit enter and your message will be sent as an SMS to the mobile number of the recepient.

Create Bot for Enterprise

  • Visit https://developer.webex.com/ then click on Start Building Apps.

  • Login with the webex teams for developers credentials.

  • Click on Create a New App.

  • Click on Create a Bot.

  • Fill the basic details of the bot such as “Bot Name”, “Bot Username”, “Icon”, “Description” then click Add Bot.

  • You will get “Bot’s Access Token”, “Bot Id”

Mapping Bot Access Token With the enterprise

Once bot is created in Webex we will get bot access token To map a Bot with an enterprise we have to call the below API or add it via UI as explained in the Adding Enterprise Cisco WebEx Console documentation.

curl -X POST https://mycompany.restcomm.com/restcomm/2012-04-24/webex/webexbot/access.json \
   -H 'Cache-Control: no-cache' \
   -H 'Content-Type: application/json' \
   -d 'accessToken=MjJjNDcxZjctZWZmYi00NGZmLWIzMzMtZWM3MWUxMjkyNzhlOGNkYjFkMWQtNjcx_PF84_fafbe78f-1599-4cb6-a070-f078ecbc0692' \
