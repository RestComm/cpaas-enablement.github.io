Number Masking Console

Overview

Number Masking enables Enterprise to easily make communication between two or more parties without revealing private information (Phone Number).

CSP can create, manage and view Enterprises. In addition, CSP can also add, manage and view Users for each Enterprise.

Both CSP Admin and Enterprise Admin can create, manage and view Applications, Contexts and Mask Pools etc. To understand more about managing Number Masking, check out the Getting Started with Enterprise Number Masking Console Guide.

This document covers the features and functionalities of the Number Masking Console (UII) from a CSP user point of view.

More information about the Number Masking API can be at the found Getting Started with the Number Masking API Guide.

To sign in to the Number Masking Console please navigate to https://<your domain>/numbermaskingconsole/.

Enterprise

Enterprises are customers of the CSP. CSPs can create, manage and delete enterprises. This is a logical grouping of CSP customers that helps to easily manage billing andinvoices for enterprises.

Enterprise List

Users can view a list of all enterprises that have been added under their organization. Enterprises can be filtered based on their name.

Enterprise Details

Users can view details of a particular enterprise by clicking on any enterprise name in the list.

Users can Delete the enterprise by clicking on the Delete Enterprise button.

Add Enterprise

Whenever the CSP adds a new enterprise, the system prompts for a mandatory eser under that enterprise. This User gets created under Restcomm as a Sub-Account of the CSP organisation. The CSP should provision all the Phone Numbers under this Sub-Account in Restcomm as explained in documentation here. These phone numbers are available for enterprises to create a Mask Number Pool for any application.

It is expected that CSP will not share newly created User credentials with Enterprise Customers. CSP should create more Users via User Management API and only share newly created credentials. This is because the first User created with Enterprise will have all the Phone Numbers provisioned and CSP wouldnt want Enterprise customers to make any changes to those numbers.

Follow the steps below to add an enterprise:

  • Click on the Add Enterprise button in the enterprise list page .

  • Enter the name of the new enterprise.

  • An enterprise admin to be assigned to this enterprise. Enter the email Id of the enterprise admin user youd want to be the admin of the enterprise. All the numbers will be provisioned under this account.

  • Enter the password for the enterprise admin account. This will be set in Restcomm.

  • Enter a friendly name of the enterprise admin user.

Enterprise name,Mail ID must be unique.
Note: Password should meet below criteria:
At least one Uppercase letter(A,B,C,D…​)
At least one Lowercase letter(a,b,c,d,…​…​.)
At least one number(1,2,3,4,5.)
At least one special character(!,@,#,$,%,&,…​…​.)
Length of the password should be greater than equal to 8.

Once the enterprise is created, click on the Applications link of a particular enterprise to see all other options mentioned as described in the Enterprise Console documentation.

Search

All the options are explained in Enterprise Console Docs. There only one more feature exposed for CSP Admin Console is CSP Admin can select the Enterprise as filter to search each of the entities.
